# 两阶段分析流水线

Workday 使用两阶段 LLM 流水线将屏幕录制转化为活动时间线。

## 第一阶段：视频转录

**输入：** 一组 15 秒视频片段（或图片帧）组成的批次

**处理：** LLM 观察屏幕内容，识别当前活动

**输出：** 3-5 条带时间戳的观察记录（Observations），例如：
- 「14:32 正在编辑 Python 文件 app.py，修改路由逻辑」
- 「14:40 浏览 Stack Overflow，搜索 async/await 相关问题」

## 第二阶段：活动卡片生成

**输入：** 第一阶段产生的观察记录

**处理：** LLM 对相近活动进行合并和总结

**输出：** 15-60 分钟粒度的活动卡片（Timeline Cards），包含：
- 活动标题和详细描述
- 开始/结束时间
- 活动分类（工作 / 学习 / 娱乐 / 其他）

## 关键参数

- `analysis.batch_size`：每批处理的视频片段数量，影响分析粒度和 Token 消耗
- `analysis.model`：分析所用的 LLM 模型 ID
- `static_frame_ratio`（视频模式）：chunk 静止率阈值，超过此值则跳过整个片段
- `static_diff_threshold`（图片模式）：相邻帧差异阈值，低于此值的重复帧被过滤

## 分析触发机制

- 后台 `AnalysisManager` 服务随应用启动自动运行
- 新产生的录制片段会自动加入分析队列
- 无需手动触发，分析在后台静默完成
