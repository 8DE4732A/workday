# 识别模式

Workday 支持两种屏幕识别模式，适配不同的视觉模型能力。

## 视频模式（默认）

将 15 秒录制片段作为视频文件整体上传给 LLM 分析。

**适用模型：** 豆包（火山引擎 ARK）、Google Gemini 等支持视频输入的模型

**静止帧处理：** 采用 chunk 级跳过策略。Workday 在提交前计算整个 15 秒片段的静止帧比例，若静止率 ≥ `static_frame_ratio`（默认 0.8），则跳过该整个 chunk，不上传也不消耗 Token。这是粗粒度去重，非逐帧比对。

**优点：**
- 模型能感知时序连续性，理解操作流程
- 上传一个文件，API 调用次数少

## 图片模式

将每帧截图单独作为图片发送，并标注时间戳。

**适用模型：** GPT-4o、Qwen-VL、LLaVA 等支持图片输入的视觉语言模型

**逐帧去重：** 相邻帧之间计算差异值，差异低于 `static_diff_threshold` 的帧会被过滤掉，只保留有内容变化的帧，减少 Token 消耗。

**时间戳标注：** 每张图片会附带录制时间戳，帮助模型理解时间顺序。

**优点：**
- 兼容性更广，不依赖视频解码能力
- 逐帧去重后 Token 消耗可控

## 批次时长的语义差异

| 模式 | 「批次时长」字段含义 |
|------|------|
| 视频模式 | 聚合多少分钟的视频片段一起分析 |
| 图片模式 | 聚合多少张图片一起分析 |

## 切换入口

在「设置 → 模型配置 → 识别模式」中选择，修改后**无需重启**即时生效。
